Key Derivation

MassaPay uses hierarchical deterministic (HD) wallets following BIP-32/BIP-39/BIP-44 standards for key generation and management.

  • 24-word mnemonic phrase (BIP-39)
  • HD derivation path: m/44'/632'/0'/0'
  • Coin type 632 for Massa blockchain
  • Hardened derivation for enhanced security

Elliptic Curve

Massa blockchain uses the secp256k1 elliptic curve, the same curve used by Bitcoin and Ethereum, ensuring battle-tested cryptography.

  • secp256k1 curve (256-bit)
  • ECDSA signatures
  • Keccak-256 for address derivation
  • Base58Check address encoding

Local Security

All sensitive data is encrypted and stored locally on your device using Android's security framework.

  • Android Keystore for key storage
  • AES-256 encryption at rest
  • Biometric authentication support
  • No cloud sync or remote storage

Network & RPC

Direct communication with Massa blockchain nodes via JSON-RPC for all operations.

  • Official Massa mainnet/buildnet nodes
  • Configurable custom RPC endpoints
  • Automatic retry with failover
  • No third-party API dependencies
